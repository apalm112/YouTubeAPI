Code Explained

Fixed Search

In the index page a form with a list of Radio Controls makes the API calls to YouTube to request videos. Each of the 7 Radio Controls shares a Uniform Resource Locator (url) to make their requests, shares an authorization key, and has a unique search string that calls for a specific programming language.

This section explains the code that supports the Radio Control form:

  1. Start Fresh

    2. The first part of the code is a JQuery function that finds to the '#html' id element as found in the web page as one of the Radio Controls. Once that form is clicked, it will run a statement to make sure that if a previous search result exists, to "emptyResults" so that only the new query results are displayed.

  2. Call the API

    3. The next part of the code prepares an API call with parameters to request videos:

    1. $.get prepares the call.
    2. A URL is specified to make the API call.
    3. 'part' describes how the data properties are delivered in the response, in this case it is comma separated list, or a "snippet".
    4. 'maxResults' displays the maximum number of videos to display.
    5. 'q' is the query string used to perform the search.
    6. 'order' organizes the videos by its 'viewCount'. Or, show me videos with the highest view count.
    7. 'key' is the authorization key to ask for a request.

  3. Display the results

    4. After the API call has been made, another function is created to gather the response data to display the video to the browser:

    1. A variable called videoOutput is created to store the data to be displayed.
    2. $.each goes through each item returned in the response.
    3. videoId returns the Video Id (the actual video in YouTube) collected from the response.
    4. videoId is then referenced by the videoOutput variable.
    5. The videoOutput variable (with the video ID) is then passed to the web browser with a JQuery function.
